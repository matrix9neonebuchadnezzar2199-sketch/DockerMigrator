コードレビュー R-2: DockerAdapter の DOCKER_HOST 尊重

2026-05-18 15:35 dev tool OK

概要

指示書 docs/instructions/code-review-r2-dockerhost-instructions.md に従い、DockerAdapter のコンストラクタを DOCKER_HOST が設定されている時は new Docker() を引数なしで呼ぶ形に変更。docker-modem の defaultOpts() が unix:// / npipe:// / tcp:// / ssh:// と DOCKER_TLS_VERIFY / DOCKER_CERT_PATH / SSH_AUTH_SOCK を自動展開する。DOCKER_HOST 未設定なら従来どおり OS 別の socketPath（Win //./pipe/docker_engine / それ以外 /var/run/docker.sock）を明示渡しして振る舞いを完全保持。execFile('docker', ...) 側（ipc/system.ts / ipc/compose.ts）は子プロセスが process.env を継承するため未変更で接続先が一致する。DockerAdapter の他メソッド・IPC チャンネル・preload・Rendererは不変。

変更ファイル

  • ~ dmig/src/main/core/DockerAdapter.ts
  • ~ docs/2026-05-18_開発日記.html

設計判断

選択肢は (A) DockerAdapter 側で DOCKER_HOST を尊重 / (B) execFile 側で env を上書きして socket 固定 / (C) DOCKER_HOST が設定されていたら拒否、の三案。マスター承認により (A) を採用。理由: Docker Desktop の context 機能やリモート daemon を意図的に使うユーザーを弾かず、docker CLI と dockerode 経由を同じ daemon に揃えるのが最も自然。TLS / SSH / 証明書まわりは docker-modem 側が完備しているため、DockerAdapter から DOCKER_HOST 以外の env を直接読むことは禁じた（Surgical Changes）。本変更でレビュー指摘 R-1 / R-2 / R-3 の三件すべて完了。
